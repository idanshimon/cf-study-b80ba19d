How do you give a chatbot a memory bigger than its brain?

Imagine you run Nimbus, a coffee subscription company. You want a support bot that knows everything — shipping, refunds, every coffee, every brewing guide. But a chatbot can only read so much text at once. Your help docs don't fit. So what do you do? This page is the answer you can click.

The problem, in one picture

A language model has a fixed reading limit per message (its "context window"). Your knowledge base does not.

❌ The naive way paste everything

Stuff all your docs into every single message. Works with 5 docs. Falls apart with 500: it's slow, it's expensive, and eventually it just doesn't fit.

✅ The ContextForge way retrieve what's needed

Store every doc on disk. For each question, pull only the 2–3 docs that actually answer it into the message. The other 9,997 stay on the shelf.

And the honest part: where it breaks

This isn't magic, it's keyword matching — so it has a real blind spot. It finds docs that share words with your question. Ask the same thing in different words, and it can whiff. Click this and watch it miss:

Why it missed: the shipping doc says "orders arrive in 3–5 business days." Your question said "show up at my door." Zero shared words → the keyword search never finds it, even though the answer is sitting right there. This is the one real limitation, and it's why you pair ContextForge with a capable model and a strict "don't guess" instruction — so when retrieval misses, the bot says "I don't have that" instead of inventing an answer.

The whole thing, in real code

Everything you just clicked is about 15 lines. This is the actual ContextForge API:

✅ With ContextForge

# Once: load your whole knowledge base
from contextforge import ContextForge

cf = ContextForge(
    db_path="nimbus.db",
    min_score=2.0,   # ignore weak matches
)
await cf.ingest("./help_docs")  # 10,000 files

# Per question: it grabs only what's needed
reply = await cf.chat("How do I get a refund?")
# → only the Returns doc enters the prompt.
# The other 9,999 stay on disk. ~150 tokens.

❌ The naive way

# Paste the entire knowledge base into
# every single message, forever:

docs = load_all("./help_docs")   # 10,000 files

So… how is this useful to you?

Swap "Nimbus coffee" for anything with more docs than fits in a prompt. Same trick, instant payoff:

  • Customer support bot — chat over your entire help center without re-pasting it every message.
  • Internal "ask the handbook" — HR policies, runbooks, wikis. Answers cite the one doc they came from.
  • Product / API assistant — hundreds of pages of docs, the user gets only the relevant slice.
  • Personal knowledge agent — years of notes on disk, each question pulls the few that matter.
The one-sentence version: ContextForge lets a chatbot act like it has read everything you own, while only ever reading the few paragraphs each question needs. Cheap, fast, and it doesn't fall over when your knowledge base grows — as long as people ask using words your docs actually contain.
